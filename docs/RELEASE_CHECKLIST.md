# Checklist de publication

Utiliser cette checklist avant de publier une nouvelle version de `ElectriX.exe`.

## Avant l'ajout au depot

- L'application se lance correctement sur Windows.
- Le nom du fichier est clair : `ElectriX.exe`.
- Le rapport PDF est a jour si le comportement de l'application a change.
- Le fichier a ete scanne avec l'antivirus local.

## Verification fonctionnelle

- L'ecran principal s'ouvre sans erreur.
- Les fonctionnalites principales ont ete testees.
- Les fichiers necessaires sont inclus dans l'executable ou disponibles dans le depot.

## Publication GitHub

- Remplacer l'ancien `ElectriX.exe`.
- Verifier que le README decrit toujours correctement le projet.
- Creer un commit avec un message clair, par exemple `Update ElectriX executable`.
- Pousser les changements sur la branche `main`.

## Apres publication

- Telecharger l'executable depuis GitHub.
- Lancer cette version telechargee pour confirmer que le fichier publie fonctionne.
